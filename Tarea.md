Proyecto: Sistema de Gestión Ganadera
El proyecto deberá evidenciar la aplicación de los conceptos abordados en la asignatura, incluyendo arquitectura cliente-servidor, consumo de APIs, manejo de bases de datos y desarrollo de interfaces web.

🎯 1. Objetivo de la actividad
Desarrollar una aplicación web funcional que permita resolver una necesidad específica, integrando tecnologías de frontend, backend y bases de datos, y aplicando buenas prácticas de desarrollo de software.

📌 2. Metodología de trabajo
Trabajo en grupos de máximo 3 estudiantes
Cada grupo debe desarrollar el problema o sistema de la lista propuesta o aprobado por el docente
El desarrollo debe realizarse durante el semestre
Todos los integrantes deben participar activamente
⚙️ 3. Requisitos del sistema
El proyecto debe cumplir como mínimo con:

🔹 Backend
Implementación de una API REST
Operaciones CRUD
Conexión a base de datos relacional
Manejo de rutas (endpoints)
Sistema de autenticación
🔹 Frontend
Interfaz web responsive
Formularios con validación
Consumo de la API
Navegación entre vistas
🔹 Base de datos
Modelo relacional
Tablas con relaciones correctamente definidas
🧩 4. Requisitos funcionales mínimos
Cada sistema debe incluir:

Gestión de al menos 5 entidades principales
Mínimo 2 roles de usuario
Registro, edición, eliminación y consulta de información
Al menos un módulo con lógica de negocio (no solo CRUD)
Visualización de información relevante (tablas, listados, etc.)
⭐ 5. Requisitos adicionales (para mejor calificación)
Dashboard con estadísticas
Reportes o consultas avanzadas
Manejo de roles y permisos
Subida de archivos o imágenes
Notificaciones
Uso de frameworks modernos (React, Vue, etc.)
📦 6. Entregables obligatorios
Cada grupo deberá entregar:

Documento del proyecto que incluya:
Nombre del sistema
Descripción
Objetivos
Funcionalidades
Roles
Requerimientos
Modelo de base de datos
Código fuente
Frontend
Backend
Script de base de datos
Repositorio (GitHub u otro)
Video o presentación del sistema funcionando
🗓️ 7. Sustentación
Cada grupo deberá presentar su sistema
Todos los integrantes deben participar
Se evaluará dominio del proyecto
Se podrán realizar preguntas individuales
🚫 8. Restricciones
No se permiten proyectos repetidos entre grupos
El sistema debe ser funcional (no solo diseño)
Se debe evidenciar integración entre frontend, backend y base de datos
El uso de herramientas de IA debe ser de apoyo, no sustituto del desarrollo
💡 9. Recomendaciones
Definir claramente roles dentro del equipo
Planificar el desarrollo por módulos
Probar continuamente el sistema
Documentar el proceso
🎯 Nota importante
El proyecto debe demostrar el desarrollo integral de un sistema web real, por lo tanto, no se aceptarán soluciones incompletas o limitadas a una sola capa (solo frontend o solo backend).
